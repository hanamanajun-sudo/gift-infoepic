## 1. 일본 상위 페이지 공통 공식 (4곳 교집합)

| 요소 | 공통 패턴 |
|------|----------|
| 리드문 | 150~250자. ①연령 심리 묘사 → ②"뭘 줄지 모르겠다" 공감 → ③해결 약속 + 근거(데이터/편집부) |
| 상품 수 | 13~16개 (필터형 카탈로그 제외). 96개 나열 아님 |
| 상품 H3 제목 | **"효익 + 상품 카테고리"** 형식. 예: 「除光液なしで落とせるはがせるマニキュア」— 상품명 나열이 아니라 '왜 좋은지'가 제목 |
| 상품당 설명 | 150~250자. 특징 1개 + 사용 장면 + 누구에게 맞는지 + 장기성("중학생 돼도 씀") |
| 선택법 섹션 | 반드시 존재. 학년/연령별 발달심리 해설 포함 |
| 예산 시세 | 반드시 명시. 일본: "평균 5,000엔" 식으로 구체적 숫자 |
| 신뢰 장치 | 최소 1개: 구매데이터(giftmall) / 학부모 122명 앙케이트(HugKum) / 편집부 엄선(MEMOCO) |
| 구매 링크 | 복수 제공 (Amazon·라쿠텐·야후 3사) |
| 전체 분량 | 3,000~5,000자 |

## 2. 사이트별 배울 점 (차별 요소)

### giftmall — 데이터 권위 모델
- "월간 700만 명 구매데이터 기반 랭킹" — 주장 대신 데이터로 권위 확보
- 리드문 3층 구조: 행동 제안 + 권위 부여 + 데이터 근거
- 선택법·시세 섹션은 기사 후반 배치
- 약점(반면교사): 상품당 설명이 제목 수준으로 얇음 → 우리는 MEMOCO 수준으로

### TANP — 필터 카탈로그 모델 (우리 /나이 /관계 /예산 페이지의 정답지)
- 6축 필터: 상황 × 관계 × 연령 × 카테고리 × 가격 × 배송
- 가격 14단계 세분화, 필터 조합마다 전용 URL → 내부링크 그물망
- 시사점: 우리 필터 페이지 구조 자체는 맞음. 콘텐츠(선택법 텍스트)가 없는 게 문제

### HugKum — 실제 목소리 모델 (신뢰의 최고 형태)
- 학부모 122명 앙케이트 + 조사 시기 명시
- 상품마다 실제 구매 부모 코멘트 인용 (지역·자녀 수까지 표기): 「본인이 갖고 싶어해서」
- 학년별(1-2/3-4/5-6학년) 발달심리 해설로 섹션 구분
- 상품 13개만 — 적게 고르는 것 자체가 큐레이션 신호

### MEMOCO — 큐레이션 카피라이팅 모델 (12歳 키워드 1위)
- H3마다 효익 선행 제목: 「とかすだけでサラツヤ髪 '電動ヘアブラシ'」
- "리얼하게 받아서 기쁜" 콘셉트를 리드문에서 선언하고 본문 전체에서 반복
- 연령 심리 언어화: 「背伸びしたい盛り」(어른 흉내 내고 싶은 시기)
- 트렌드 구체성: K-POP, 롬앤(한국 브랜드) 언급 — 일본조차 한국 트렌드 인용
- 장기성 소구 일관: "중학생 되어도 쓸 수 있다"

## 4. 우리만의 차별화 (일본 상위권에도 없는 것)

1. **안티 추천 섹션** — "사지 말아야 할 것 3가지". 4곳 모두 없음. 신뢰 신호 최강.
2. **받는 아이 유형 분기** — "게임 좋아하면 A / 만들기 좋아하면 B / 꾸미기 좋아하면 C" 의사결정 트리
3. **실패담 기반 팁** — 큐레이션 승인 세션에서 나온 실제 판단 한 줄을 본문에 반영

## 5. 최종 본문 템플릿 (가이드당)

```
[리드문 200자] 연령 심리 → 고민 공감 → "쿠팡 리뷰 데이터 + 직접 선별" 근거 제시

## {연령·관계}가 진짜 좋아하는 선물 TOP 5~7        ← Products DB 렌더 위치
  ### {효익} "{상품 카테고리}"                      ← 효익 선행 제목
      상품당 200자: 특징 + 사용 장면 + 이런 아이에게 + 리뷰수·평점
      (Products DB: 쿠팡 파트너스 링크 + 네이버 링크)

## 가격대별로 고르기
  ### 3만원대 최선 / ### 5만원대 최선 / ### 10만원대 최선

## {연령·관계} 선물 고르는 법
  발달심리·취향 해설 + 받는 아이 유형 분기(A/B/C)

## 이건 사지 마세요 (실패 사례 3가지)               ← 차별화 섹션

목표 분량: 1,500~2,500자 (표 1개 이상, 점프링크 TOC 자동)
```

## 6. 작업 파이프라인 (반복 실행용, 2026-07-03 실측 반영)

1. **여론 수집(자동, 구현됨)**: `npm run collect -- "키워드"` → `scripts/research/collect.mjs`
   - 네이버 블로그 검색 API(`naver-blog.mjs`) + 유튜브 검색/댓글 API(`youtube.mjs`) → `src/data/research/{키워드}.json`
   - **실측 노이즈율**: 블로그 20건 중 협찬·판매성 글이 절반. 유튜브는 "쿠팡 추천템" 광고 쇼츠가 섞이면 댓글이 전부 구매문의라 무신호 — "찐으로 받고 싶은 선물" 류 **위시리스트/댓글모음 포맷**이 신호 밀도가 압도적으로 높았음. 키워드에 "후기", "찐", "받고싶은" 등을 넣어 광고 쇼츠 유입을 줄일 것
2. **일본 커뮤니티 교차검증(수동, ad-hoc)**: 무료 API 없음(知恵袋 API는 2017년 유료화). WebSearch로 `{연령} {성별} 誕生日プレゼント 知恵袋` 검색 → 관련 스레드 2~3개 WebFetch로 읽고 상품 카테고리·실패담 추출
   - 한일 교차 검증되는 카테고리는 신뢰도 상승 신호로 취급
   - 브랜드명은 이식 금지(한국 대체재로 치환), **금액은 맥락 확인 필수**(생일선물 vs 사은품 예산은 자릿수가 다름)
3. **상품 교차매칭(자동)**: 여론 수집에서 뽑은 카테고리를 네이버쇼핑 API로 재검색 → 실제 가격·판매처·링크 확보 (쿠팡 API는 리뷰수·평점 필드 없음 — 자동 필터링 불가, 최종 후보만 수동 확인)
4. **큐레이션(승인 세션)**: Claude가 후보 5~7개 + 근거(커뮤니티 인용) 표로 정리 → **사용자 승인·교체·한줄 코멘트**
5. **본문 작성**: 위 템플릿 5절 구조로 작성 → Notion 본문 교체
6. **Products DB 입력**: ProductCard + ItemList JSON-LD 자동 활성화
7. **우선순위**: GSC 키즈 클러스터 12~14개 먼저 (12세여아 → 13세여아 → 10세남아 → 10-12세남아 → …)
